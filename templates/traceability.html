Genealogy Complexity Notes

Lot traceability in wire & cable manufacturing is inherently complex due to several edge cases that this system handles via recursive Common Table Expressions (CTEs) with a depth limit of 10 tiers:

  • Circular references (rework loops): A defective lot may be reworked and re-enter the process as input to a new lot, creating a cycle. The depth limit prevents infinite recursion.
  • Partial consumption: A single input lot (e.g., compound batch CB-0330) may be split across multiple output lots on different extrusion lines, creating a one-to-many fan-out.
  • Blended lots: Multiple input materials (copper, insulation, shield) combine into a single cable lot — a many-to-one convergence requiring backward trace through all branches.
  • Multi-tier depth: Raw material (copper rod) → drawn wire → insulated conductor → shielded assembly → jacketed cable → cut lengths → reels. Each tier is a separate lot_tracking record.
  • Cross-WC genealogy: A lot passes through 8-10 work centers (COMPOUND → DRAW → CV → BRAID → CABLE → JACKET → TEST → CUT), each creating new lot relationships.
  • Splice zone ownership: When conductor payoffs are spliced on the stranding line, the splice zone (typically 2-5 feet) inherits properties of both parent lots. For MIL-SPEC products, splice zones are marked, documented, and may require customer acceptance. Our model tracks splice events as special lot_tracking records with tier_level='Splice'.
  • Regrind/blend-back tracing: Scrap PVC compound is reground by weight and blended into new insulation batches (max 10% regrind per ASTM D2287). Genealogy must track mass fraction: if Lot A contributes 8% regrind to Lot B, and Lot B is later recalled, Lot A's source material must be traceable. Our lot_tracking model supports this via qty_consumed (weight-based) + tier_level='Regrind'.
  • Remnant inheritance: When a 1,000 ft reel is cut to customer order (e.g., 950 ft shipped + 50 ft remnant), the remnant lot inherits all quality certifications and test results from the parent. Remnants below minimum order quantity are downgraded to stock or scrapped. Our reel_inventory tracks remnants via status='Remnant' with parent reel reference.

Performance target: Full backward trace from finished reel to raw material in <10 seconds. Current implementation uses SQLite recursive CTEs with indexed output_lot and input_lot columns.
